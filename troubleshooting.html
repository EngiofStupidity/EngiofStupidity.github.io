  • If some clients are connecting and others are not, check if you've enabled network booting on them. You can easily check by plugging an ethernet cable into the Pi without inserting an SD card. The LEDs on the Pi and the connector should turn on. If they don't, see the clients section of the how to install page.
  • If you can't connect any clients, check if the network is suitable:
    1. Format an SD card and copy bootcode.bin from a standard Raspbian Image on it
    2. Plug the SD card into a client Pi and check if it appears as a new MAC address in the PiServer GUI.
    3. If it appears, this is a known issue, and you can head to the PiServer forums for help.

    For a temporary fix, clone the SD card with bootcode.bin for all the clients.

If you have any more issues you can head to the PiServer forums or the GitHub Issues page.
